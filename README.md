# 💰 간단 가계부 웹 애플리케이션

## 📌 프로젝트 소개

AI 에이전트를 활용하여 개발한 간단한 가계부 웹 애플리케이션입니다.

사용자는 거래 내역을 기록하고 월별 소비 흐름을 확인할 수 있으며,

React 기반 프론트엔드와 Supabase 기반 백엔드를 활용해 구현했습니다.

또한 Docker와 Supabase CLI를 이용해 로컬 개발 환경 및 데이터베이스 마이그레이션 환경을 구성했습니다.

## 기능 정책

1. 회원 인증 정책 (Authentication)
    - 인증 방식: Supabase Auth를 이용한 이메일/비밀번호 기반 인증.
    - 데이터 보안 (RLS): Row Level Security 정책을 적용하여, 사용자는 본인이 생성한 데이터만
    조회/수정/삭제할 수 있음.
    - 세션 관리: 로그인 성공 시 세션을 유지하며, 비로그인 사용자는 인증 화면으로 강제 리다이렉트됨.
2. 카테고리 관리 정책 (Categories)
사용자의 소비와 수입을 분류하는 핵심 시스템입니다.
    - 카테고리 유형: '지출(Expense)'과 '수입(Income)'으로 엄격히 구분됨.
    - 이름 중복 방지:
        - 동일한 사용자 내에서, 동일한 유형(지출/수입) 하에 같은 이름의 카테고리 생성이 불가능함 (DB
        제약 조건 적용).
        - 단, 지출 카테고리에 '기타'가 있더라도 수입 카테고리에 '기타'를 만드는 것은 허용됨.
    - 시각적 요소 (Style):
        - 색상(Color): 12종의 고정 HEX 컬러 팔레트를 제공하여 시각적 일관성 유지.
        - 아이콘(Icon): 20종의 Lucide 아이콘 라이브러리 제공.
        - UI 표현: 아이콘 자체에 색상을 입히지 않고, 아이콘 왼쪽에 별도의 '색상 바(Color Bar)'를
        배치하여 가독성 증대.
    - 권한 체계:
        - 기본 카테고리: 시스템이 기본 제공하며 모든 사용자가 공통으로 사용 (수정/삭제 불가).
        - 개인 카테고리: 사용자가 직접 생성하며, 본인만 확인 및 관리 가능 (수정/삭제 자유).
3. 거래 내역 관리 정책 (Transactions)
일일 지출 및 수입을 기록하는 기능입니다.
    - 필수 입력 항목: 금액, 유형(지출/수입), 카테고리, 날짜.
    - 금액 처리:
        - 지출은 데이터상 음수/양수 여부와 상관없이 UI에서 빨간색(-)으로 표시.
        - 수입은 파란색(+)으로 표시.
    - 카테고리 연동: 거래 내역 작성 시 선택한 유형(지출/수입)에 맞는 카테고리 목록만 필터링되어 노출됨.
    - 실시간 동기화 (Realtime):
        - 데이터베이스 변경 사항이 대시보드와 목록에 즉시 반영됨 (새로고침 불필요).
    - 삭제 정책: 거래 내역 삭제 시 즉시 DB에서 제거되며, 연결된 카테고리에는 영향을 주지 않음.
4. 대시보드 및 리포트 정책 (Dashboard)
    - 월별 필터링: 사용자가 선택한 '년-월' 기준의 데이터만 합산하여 표시.
    - 통계 지표:
        - 총 수입: 해당 월의 모든 수입 합계.
        - 총 지출: 해당 월의 모든 지출 합계.
        - 현재 잔액: 총 수입 - 총 지출 계산값.
    - 시각적 요약: 수입과 지출의 비율을 프로그레스 바 형태로 시각화하여 현재 소비 상태를 직관적으로
    파악 가능.
5. 데이터베이스 무결성 정책
    - 참조 무결성: 카테고리가 삭제될 때 해당 카테고리를 참조하는 거래 내역이 있다면 삭제가 제한되거나
    보호됨 (외래키 제약).
    - 초기화 지원: 시스템 장애나 데이터 오염 시 특정 사용자의 내역만 안전하게 초기화할 수 있는 구조
    설계.

## 트러블 슈팅

### 문제 상황

github에 작업 내용들을 정리해서 push 하는 과정에서 빈번한 오류 발생

### 원인

작업 내용을 정리하는 과정에서 에이전트가 git log를 최근 로그 일부분만 읽고 작업하는 경우 발생, 반복된 오류 수정 시도로 git log가 더 지저분해짐.

### 해결 방법

히스토리 정리용 branch를 새로 만들어서 git log를 모두 읽고 작업 내용을 단계별로 나누어서 정리, commit과 push 전에 다시 한 번 파일 목록들과 내용을 체크
