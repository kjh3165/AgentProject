# 카테고리 관리 기능 고도화 설계 (Spec)

- **작성일**: 2026-05-14
- **상태**: Draft (승인 대기)
- **주요 목표**: 사용자가 카테고리를 직접 관리(CRUD)하고, 시각적 요소(색상, 아이콘)를 부여하여 가계부 데이터를 직관적으로 파악할 수 있게 함.

## 1. 데이터베이스 변경 사항

### `categories` 테이블 스키마 확장
- `color`: `TEXT` (Tailwind CSS 색상 클래스 또는 Hex 코드, 예: `text-red-500`)
- `icon`: `TEXT` (Lucide-React 아이콘 식별자, 예: `ShoppingBag`)

### 마이그레이션 전략
- 기존 기본 카테고리(식비, 교통비 등)에 어울리는 색상과 아이콘을 배정하는 `UPDATE` 쿼리 포함.
- 신규 필드는 `NOT NULL` 제약 조건 없이 추가한 뒤 기본값을 부여하거나, 적절한 기본값(예: `gray`, `Tag`)을 설정.

## 2. 신규 페이지 및 UI 구성

### `/categories` (카테고리 관리 페이지)
- **목록 뷰**: 수입/지출 탭으로 구분하여 현재 카테고리 목록 표시.
- **시각화**: 각 항목 옆에 선택된 색상과 아이콘을 미리보기로 표시.
- **CRUD**:
    - **추가/수정 모달**: 이름(Input), 유형(Select), 색상(12가지 팔레트), 아이콘(20~30개 그리드) 선택 기능.
    - **삭제**: 해당 카테고리를 사용 중인 거래 내역이 있을 경우 경고 알림 표시.

### UI 요소 상세
- **색상 팔레트**: 디자인 일관성을 위해 엄선된 12가지 색상 제공 (예: Red, Orange, Amber, Emerald, Blue, Indigo, Violet, Rose 등).
- **아이콘 라이브러리**: Lucide-React의 주요 가계부 관련 아이콘 (Utensils, Car, Home, ShoppingCart, Heart, Smartphone, Gift, Wallet 등).

## 3. 기존 기능 연동

### `TransactionForm` & `TransactionList`
- 카테고리 선택 드롭다운 및 목록 표시 시, 텍스트 옆에 해당 카테고리의 아이콘과 색상을 함께 렌더링.

## 4. 테스트 전략
- 카테고리 추가/수정/삭제 기능 정상 작동 여부 확인.
- RLS 정책에 따라 본인의 카테고리만 수정/삭제 가능한지 검증.
- 카테고리 변경 사항이 실시간(Realtime)으로 목록에 반영되는지 확인.
