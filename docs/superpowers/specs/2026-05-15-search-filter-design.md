# 설계 사양서: 거래 내역 검색 및 상세 필터링 (기본형)

## 1. 개요
사용자가 현재 선택된 월 내의 거래 내역을 키워드, 유형, 카테고리별로 필터링하여 원하는 기록을 빠르게 찾을 수 있도록 돕는 기능을 구현한다.

## 2. 주요 기능 및 UI 구성
### 2.1 아코디언형 필터 섹션 (UI)
- **위치**: `Transactions.tsx` 페이지 상단, 내역 리스트 바로 위.
- **트리거**: '필터' 아이콘 버튼 클릭 시 토글(펼침/닫힘).
- **구성 요소**:
    - **검색창**: `description` 필드 대상 키워드 입력.
    - **유형 필터**: 3단 전환 버튼 (전체 / 지출만 / 수입만).
    - **카테고리 필터**: 현재 유저의 카테고리 목록을 보여주는 드롭다운(Select).
    - **액션 버튼**: '검색 적용' 버튼 및 '초기화' 버튼.

### 2.2 검색 및 필터링 로직 (Data)
- **범위**: 현재 선택된 월(`selectedMonth`) 내의 데이터로 한정.
- **실행 타이밍**: '검색 적용' 버튼 클릭 시 또는 Enter 키 입력 시.
- **쿼리 방식 (Supabase)**:
    - 키워드: `.ilike('description', `%${keyword}%`)`
    - 유형: 선택된 경우 `.eq('type', type)`
    - 카테고리: 선택된 경우 `.eq('category_id', categoryId)`

## 3. 구현 단계 (다음 세션 예정)
1. `Transactions.tsx`에 필터 상태값 및 토글 UI 추가.
2. `TransactionList.tsx`에서 필터 조건을 props로 받아 Supabase 쿼리에 동적 적용.
3. 필터 초기화 및 예외 처리(검색 결과 없음 등) UI 구현.
